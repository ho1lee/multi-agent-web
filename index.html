1. 과제 수행 배경과 세미나 목표

과제 수행 배경

Palantir AIP 도입 전 투자의 타당성과 효과성을 검증하기 위해, 현재 DS Agent 개발 환경에서 Multi-Agent를 구현하고 기능 및 효과를 비교 검토합니다.

  • 기존 시스템과의 통합 비용 파악
  • 운영 규칙 및 RBAC 정책 사전 점검
  • 관측성 확보를 위한 로그·메트릭·트레이싱 설계

세미나 목표

참석자가 LLM, Agent, Multi-Agent, MCP, A2A에 대한 개념을 이해하도록 돕습니다.

  • 용어 정의와 관계 구조 파악
  • 운영 관점에서의 장점과 주의사항
  • 장애 대응과 비용 예측 방법

2. 개념

대규모 언어 모델은 많은 문장을 학습해 언어 패턴을 예측합니다. 초급자에게는 커다란 자동 완성 도구처럼 느낄 수 있습니다.

  • 배포 시 모델 크기에 따른 비용 변동
  • 환각 가능성에 대비한 검증 프로세스
  • 로그와 감사 기능으로 책임 추적

Agent는 LLM 위에 규칙과 도구 사용 능력을 얹어 특정 업무를 수행하도록 만든 모듈입니다. 운영자의 스크립트 실행 자동화처럼 이해할 수 있습니다.

  • 권한 부여(RBAC)로 실행 범위 제한
  • 툴 실패 시 재시도 및 롤백 절차 필요
  • 메트릭 기반 성능 모니터링

Model Context Protocol은 서로 다른 시스템이 공통 방식으로 LLM과 정보를 주고받도록 하는 규약입니다.

  • 표준화된 인터페이스로 통합 비용 감소
  • 데이터 접근 로그 남김으로 감사 용이
  • 장기 컨텍스트 손실 시 재동기화 필요

여러 Agent가 역할을 나누어 협업하면 한 Agent가 처리하기 어려운 문제를 나눠 해결할 수 있습니다.

  • 단일 LLM의 한계점: 긴 업무에서 정보 손실
  • 작업 분산을 통한 성능 향상
  • 전문화된 역할 분담으로 장애 격리

Agent-to-Agent 상호작용은 여러 Agent가 서로 메시지를 주고받아 협업하는 방식을 말합니다.

  • 권한 분리로 보안 경계 강화
  • 통신 실패 시 재시도 및 알림
  • 비용 집계로 최적화 지점 파악
